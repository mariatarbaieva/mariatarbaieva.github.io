Data Modelling

Data structure repository. Here you create a new extracts, set up extracts parameters

for correct extraction from the document.

Problem

In order to establish self-service business needs to enable customers to create data models themselves in an easy and intuitive way. Data scientists already had their interface, but to reach proper data structure they are using excel sheets and writing queries to find and sort information. There are a lot of frictions between customers --> Technical Consultants --> Data scientists. Data points should be reused to perform proper extraction. That's, why interface where these 3 user groups can interact and use as one source of truth was not only nesecesity but business requirement as well.

User research

Together with Product manager we conducted user interviews, had learning sessions with Data scientists and technical consultats to understand their workflows and jobs to be done.

David

Technical consultant
- Provide clients with data model that matches with requirements for all their documents;
- Quickly navigate to data fields with an additional information;
- To use existing extracts that perform well in the data model as much possible

Julie

Data scientist
- Structure and set up extract properties;
- Analyse and compare data in the context;
- Restrict permissions for customers to modify extract properties that will affect AI performance

Neal

Asset Manager
- Create project data model asap using templates;
- Manage projectâ€˜s DM in an intuitive way with clear expectations of the behaviour

Learnings

Based on user interviews and sessions with stakeholders, I outlined user flow across applications that are part of the product to better visualize the role of application we are going to build.

Objects

In order to aplly all knowledge we learned and help stakeholders extract nessecary data they keep in mind as well as help our team to undestand application better we organised crossfunctional workshop with data scientists, technical consultants and frontend, backend developers where we used object oriented design technique.

Result

Improved workflows where concistent with other applications and brought transparency in data modelling process so Technical consultants could onboard customer users more smoothly.
